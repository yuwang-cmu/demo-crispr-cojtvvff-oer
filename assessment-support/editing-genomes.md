# Chapter 2 Assessment Guide: Editing Genomes

> **Demonstration only:** This AI-generated Coursewerk guide is not intended for direct use in actual teaching, laboratory work, or clinical decisions.

## Source and format

Adapted and reorganized from the revision-bound Wikipedia sources in `metadata/SOURCE_RECORD.json` under CC BY-SA 4.0. This is assessment-design guidance, not a secure question bank or operational protocol.

## General assessment priorities

- Require learners to separate intended change, molecular event, repair outcome, measured evidence, and interpretation.
- Use allele-frequency tables, assay-limit scenarios, control design, and claim revision.
- Never grade “most efficient” or “safest” without a defined cell type, target, delivery context, and assay.
- Keep empirical and normative questions distinguishable in governance tasks.

## Objective 1: Separate the stages of an editing workflow

### Target understanding

Learners order question, intended outcome, target/editor design, delivery, molecular action, repair or conversion, validation, and interpretation.

### Question guides

#### Guide A — Workflow reconstruction
- **Variables:** 7–10 shuffled stages; rotate break, base-edit, and dCas9 contexts.
- **Constraint:** each arrow needs a causal or decision verb.
- **Worked instantiation:** intended substitution → editor design → delivery → locus conversion → allele quantification → function assay.

#### Guide B — Missing-stage critique
- **Variables:** short methods-and-claim narrative with one omitted stage.
- **Constraint:** identify the missing evidence and the overclaim it causes.
- **Worked instantiation:** phenotype after delivery without locus sequencing cannot establish that the intended edit occurred.

|---|---|---|
| Homework | Apply | Reconstruct two workflows. |
| Quiz | Understand | Order stages. |
| Exam | Analyze | Diagnose a skipped link. |
| Project | Create | Draft an evidence-chain plan. |

## Objective 2: Predict repair outcome distributions

### Target understanding

Learners compare end joining, MMEJ, and template-supported repair while recognizing precise joins, indels, mixed alleles, and unedited cells.

### Question guides

#### Guide A — Allele-table interpretation
- **Variables:** 3–8 alleles totaling 100%; include reference and intended outcome.
- **Constraint:** report identities and frequencies, not one collapsed efficiency.
- **Worked instantiation:** 55% intended, 25% frameshift, 10% in-frame, 10% reference is not simply “90% success.”

#### Guide B — Pathway prediction
- **Variables:** template present/absent, cell-cycle context, microhomology, end structure.
- **Constraint:** use “more/less plausible,” not deterministic language.
- **Worked instantiation:** a donor template and compatible cell state make template-supported repair more plausible but do not guarantee it.

|---|---|---|
| Homework | Analyze | Interpret allele distributions. |
| Discussion | Evaluate | Debate what counts as success. |
| Quiz | Apply | Match outcomes to repair logics. |
| Exam | Analyze | Compare two cell contexts. |

## Objective 3: Compare delivery trade-offs

### Target understanding

Learners compare DNA, mRNA/guide, and RNP delivery by onset, duration, cargo, uptake, cell stress, and experimental scale rather than ranking them universally.

### Question guides

#### Guide A — Contextual selection
- **Variables:** cell type, desired duration, cargo size, repeat dosing, viability.
- **Constraint:** choose a method and name at least two trade-offs.
- **Worked instantiation:** RNP may suit brief exposure, but uptake and viability still require testing.

#### Guide B — Confounding diagnosis
- **Variables:** delivery-only control, editor control, mock treatment, time course.
- **Constraint:** identify a phenotype that could arise from delivery stress.
- **Worked instantiation:** reduced cell growth in both editor and mock-electroporation groups weakens an edit-specific explanation.

| Purpose | Cognitive level | Design |
|---|---|---|
| Homework | Apply | Select delivery for three contexts. |
| Quiz | Understand | Match form to exposure profile. |
| Exam | Evaluate | Diagnose delivery confounding. |
| Project | Create | Build a trade-off decision matrix. |

## Objective 4: Distinguish quality dimensions

### Target understanding

Learners distinguish on-target efficiency, on-target heterogeneity, off-target activity, functional effect, and toxicity, and select assays appropriate to each.

### Question guides

#### Guide A — Metric-to-assay match
- **Variables:** amplicon sequencing, long-read assay, candidate-site assay, protein activity, phenotype, viability.
- **Constraint:** state what each assay cannot establish.
- **Worked instantiation:** short amplicons quantify local alleles but may miss larger rearrangements.

#### Guide B — Dashboard critique
- **Variables:** 4–6 reported metrics with one missing category.
- **Constraint:** learner identifies the unsupported conclusion.
- **Worked instantiation:** high on-target efficiency plus normal viability does not establish genome-wide specificity.

| Purpose | Cognitive level | Design |
|---|---|---|
| Homework | Analyze | Map assays to claims and blind spots. |
| Discussion | Evaluate | Audit a result dashboard. |
| Quiz | Apply | Choose the appropriate metric. |
| Exam | Evaluate | Write a bounded result statement. |

## Objective 5: Design layered validation

### Target understanding

Learners connect delivery, locus sequence, allele distribution, molecular function, phenotype, and causality using controls, replication, and orthogonal evidence.

### Question guides

#### Guide A — Evidence ladder
- **Variables:** provide 2–5 completed layers and ask for the next two.
- **Constraint:** every added assay must answer a named alternative.
- **Worked instantiation:** after confirming locus sequence, measure protein function and use an independent guide or rescue to test causality.

#### Guide B — Control selection
- **Variables:** mock delivery, non-targeting guide, independent guide, rescue, untreated, positive control.
- **Constraint:** match each control to a confound.
- **Worked instantiation:** mock delivery tests whether the procedure rather than targeting causes the phenotype.

| Purpose | Cognitive level | Design |
|---|---|---|
| Homework | Create | Build a validation ladder. |
| Quiz | Apply | Match control and confound. |
| Exam | Evaluate | Identify the strongest missing evidence. |
| Project | Create | Pre-register a conceptual validation plan. |

## Objective 6: Compare governance contexts

### Target understanding

Learners distinguish technical evidence from judgments about consent, benefit, justice, reversibility, propagation, and participation across somatic, heritable, agricultural, and ecological editing.

### Question guides

#### Guide A — Same editor, different context
- **Variables:** identical molecular performance across two deployment settings.
- **Constraint:** compare at least three governance dimensions.
- **Worked instantiation:** ex vivo somatic editing and heritable embryo editing differ in consent, monitoring, and intergenerational effects.

#### Guide B — Empirical/normative sorting
- **Variables:** 8–12 questions from a policy discussion.
- **Constraint:** identify interactions without collapsing categories.
- **Worked instantiation:** “How often does the edit spread?” is empirical; “What spread is acceptable?” is normative and informed by evidence.

| Purpose | Cognitive level | Design |
|---|---|---|
| Homework | Analyze | Sort and connect question types. |
| Discussion | Evaluate | Map affected stakeholders. |
| Exam | Evaluate | Compare two governance settings. |
| Project | Create | Draft an inclusive decision framework. |

## Rubric themes

1. The workflow does not skip cellular repair or measurement.
2. Outcome distributions are reported without collapsing unlike alleles.
3. Assay claims remain within resolution and sampling limits.
4. Controls are linked to explicit alternative explanations.
5. Ethical conclusions distinguish evidence, values, stakeholders, and context.
